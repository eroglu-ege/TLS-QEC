## UNDERSTOOD

### Initialization vs active reset
- Initialization: set state once at t=0, evolve freely
- Active reset: every dt, force TLS back to 1, leave qubit, evolve again
- Active reset models TLS continuously re-excited by environment
- Qubit shows staircase buildup toward P_ss over N cycles
- TLS shows sawtooth: reset to 1, drain to some P_t_end, reset again

### Ensemble results
- 30 random P_q^(0) values, same seed in both 17 and 18
- Memory erasure: both protocols drive P_q toward P_ss = 0.083
- Active reset keeps qubit at higher effective population than free evolution
  (TLS keeps injecting energy every dt)
- Correlation(final P_q, initial P_q^(0)) measures memory retention
- Std of ensemble collapses over time as system thermalizes

### TLS lifetime physics
- TLS decays ONLY via qubit (no direct bath): reversed Purcell effect
  Gamma_TLS(Delta) = 2g^2 * (1/T2_q) / (Delta^2 + (1/T2_q)^2)
- 1/T2_q is the correct qubit linewidth (not 1/T1_q) — includes dephasing
- At Delta=0: T1_TLS = T2_q / (2g^2) — minimum lifetime
- At large Delta: T1_TLS -> 1/gamma_t_int (unknown saturation plateau)
- Setting g = gamma_phi puts system at Solomon threshold

### Key counterintuitive result — Sweep 2
  T1_TLS ~ T2_q: better qubit (longer T2) = LONGER TLS lifetime
  Physical reason: narrower qubit linewidth = less spectral overlap with TLS
  = slower TLS drain. Improving qubit coherence does NOT help at resonance —
  it actually allows the TLS to stay excited longer and cause more errors.

### Key experimental result — Sweep 5
  Qubit T1_eff dips at resonance with TLS — this is what swap spectroscopy
  measures. Width of dip -> gamma_t_TLS. Depth of dip -> g.
  Lindblad and Solomon disagree at g ~ gamma_phi (our threshold region) —
  direct experimentally falsifiable prediction from this thesis.

### Temperature dependence — Sweep 3
  Thermal broadening: gamma_q_eff = gamma_q*(2*n_th+1)/2 + gamma_phi
  Warmer fridge -> broader qubit linewidth -> faster TLS drain
  But also: more thermal errors. Trade-off not captured by Solomon.

### g ~ Gamma_2phi connection
  gamma_phi = 1/T_phi = 2/T2 - gamma_q
  Setting g ~ gamma_phi means: coupling comparable to pure dephasing rate
  This is exactly the Solomon breakdown threshold we found numerically
  (g/gamma_t ~ 0.5-2) because gamma_phi ~ gamma_t/2 in our parameters

## PHYSICS CONCEPTS DISCUSSED
- Transition frequency omega_q vs Rabi frequency Omega vs vacuum Rabi g
  omega_q = energy gap (GHz, qubit property)
  Omega   = drive Rabi rate (MHz, set by microwave power)
  g       = vacuum Rabi rate (MHz, set by coupling geometry)
  Analogy: omega_q = pitch, Omega = how hard you strike, g = coupling to cavity
- Purcell effect: enhanced decay when resonant with leaky cavity
  gamma_P = 4g^2/kappa at resonance
  Lorentzian in detuning: drops 100x for Delta = 5*kappa
- Semigroup breakdown in NMOQS: CP-divisibility fails during Rabi intervals
  Information backflow from TLS to qubit during coherent exchange
  Promoting TLS to system (not bath) makes it Markovian again
- Kraus representation: valid when initial state factorizes + Markov holds
  Unitary on full universe -> partial trace -> irreversible semigroup
